# **UI/UX Specification: Digital Arena (Modernized)**

## **1\. Visual Identity**

* Primary Palette: \#00A3FF (Digital Blue), \#FF9100 (Evolution Orange), \#121212 (Deep Slate/Background).  
* Style: Glassmorphism with neon borders.  
* Typography: Bold, monospaced fonts for stats (e.g., "Orbitron" or "Roboto Mono").

## **2\. Screen Layout (16:9 Landscape)**

### **A. The Field (Layer: GameField)**

* Perspective: 2.5D "Tabletop" tilt. The field is a trapezoid shape to simulate depth.  
* Center-Left (Enemy Active): Large Card Sprite with a dynamic shadow underneath.  
* Center-Right (Player Active): Large Card Sprite, slightly angled toward the camera.  
* The "Slot" Zones:  
  * Support Slot: A glowing rectangular frame behind the Active Digimon.  
  * Evolution Pile: A stacked card graphic showing the "History" of the current Digimon.

### **B. The Player HUD (Layer: UI\_Overlay)**

* Bottom-Left (Player Stats):  
  * HP Bar: A long, segmented bar (Cyber-style). Fills with Green/Yellow/Red.  
  * DP Counter: Large hexagonal icon with a pulsing number.  
* Bottom-Center (The Hand):  
  * Cards are arranged in a slight "Fan" arc.  
  * When hovered, a card moves \+30px Y-axis and scales by 1.2x.  
* Top-Right (Enemy Stats): Mirrored version of the Player HUD but smaller and "Red-Themed" to indicate hostility.

### **C. The Command Hex (Layer: UI\_Interactions)**

* A floating hexagonal menu that appears during the Attack Phase.  
  * O (Top-Left): Red Neon.  
  * △ (Top-Right): Green Neon.  
  * X (Bottom): Blue Neon.

## **3\. Technical Implementation (C3 Architecture)**

### **Layer Hierarchy (Bottom to Top)**

1. Background\_3D: Static or parallaxing "Cyber Space" background.  
2. Field\_Geometry: The 2.5D tilted table. (Uses a 'Perspective' effect on the layer).  
3. Card\_Layer: Where Fam\_Cards instances live.  
   * *Z-Order Tip:* Cards in hand have a higher Z-Elevation than cards on the field.  
4. FX\_Layer: Additive blend mode for "Digivolving" light beams and "Attack" slashes.  
5. HUD\_Static: HP/DP bars and Deck counters. (Parallax 0,0).  
6. HUD\_Interactive: The Command Hex and Support Card reveal animations.

### **Construct 3 Specific Logic**

* The "Fan" Hand Logic:  
  * Use a loop: For i from 0 to Hand.Count.  
  * Set X to (ViewportWidth/2) \+ (i \- Hand.Count/2) \* Spacing.  
  * Set Angle to (i \- Hand.Count/2) \* ArcIntensity.  
* The UI Glass Effect:  
  * Create a 9-patch sprite with a semi-transparent center and a bright 2px border.  
  * Apply the Blur effect to the Background\_3D layer only when a Modal (like "Support Phase") is active.  
* Responsiveness:  
  * Use the Anchor Behavior for all HUD elements.  
  * Player HP → Anchor Left/Bottom.  
  * Enemy HP → Anchor Right/Top.
