# **Engineer Implementation Guide: UI & Card Interactions**

## **1\. Card State Machine**

Every card in the hand must have an Instance Variable State.

* State \= 0 (In Deck): Invisible, collision disabled.  
* State \= 1 (In Hand): Reacts to Mouse Hover (Tween Scale).  
* State \= 2 (Playing): Trigger "Move to Field" Tween.  
* State \= 3 (Active Field): Pinned to Field Slot.

## **2\. Dynamic Data Binding**

Do not create text objects inside the Card Sprite.

* **The "Container" Method:** Create a Card Sprite and a "Card\_Text\_Family".  
* On Start of Layout, for each Fam\_Cards:  
  * Create Txt\_Name, Txt\_HP, Txt\_Atk.  
  * Use the Pin behavior to lock them to the card.  
  * Update their content using Cards.JSON based on CardID.

## **3\. Z-Sorting Logic**

To ensure the card being dragged or hovered is always on top:

Condition: Fam\_Cards \-\> On Mouse Over  
Action: Fam\_Cards \-\> Move to top of layer  
Action: Fam\_Cards \-\> Tween "Hover" (Scale 1.1, 0.1s)

## **4\. Performance Optimization (UI)**

* **Layer Compositing:** Keep the Background\_3D and Field\_Geometry on their own layers. Set these layers to Force Own Texture \= No unless using specialized shaders to save draw calls.  
* **Sprite Fonts:** Use Sprite Fonts instead of standard Text objects for HP/DP numbers to ensure the "Digital" look remains crisp across all resolutions without rendering overhead.
